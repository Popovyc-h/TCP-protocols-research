# Топ 20 портів

Порт | Протокол | Сервіс | Опис
--- | --- | --- | ---
20, 21 | TCP | FTP | File Transfer Protocol — передача файлів
22 | TCP | SSH | Secure Shell — безпечне віддалене керування командним рядком
23 | TCP | Telnet | Telnet — незашифрований віддалений доступ (застаріває)
25 | TCP | SMTP | Simple Mail Transfer Protocol — відправлення пошти
53 | TCP/UDP | DNS | Domain Name System - перетворення доменних імен на IP-адреси
67/68 | UDP | DHCP | Dynamic Host Configuration Protocol - автоматична видача IP-адрес пристроям
69 | UDP | TFTP | Trivial FTP - простий протокол передачі файлів (без автентифікації)
80 | TCP | HTTP | Звичайний, незашифрований вебтрафік
88 | TCP | Kerberos | Автентифікація користувачів (часто використовується в Active Directory)
110 | TCP | POP3 | Post Office Protocol - отримання пошти (з локальним завантаженням)
123 | UDP | NTP | Network Time Protocol - синхронізація точного часу на пристроях
143 | TCP | IMAP | Internet Message Access Protocol - отримання пошти (зі зберіганням на сервері)
389 | TCP | LDAP | Lightweight Directory Access Protocol - пошук та керування інформацією про користувачів у мережі
443 | TCP | HTTPS | Безпечний, зашифрований вебтрафік (SSL/TLS)
445 | TCP | SMB | Server Message Block - спільний доступ до файлів та принтерів у мережі Windows
631 | TCP | IPP | Internet Printing Protocol - мережевий друк
993 | TCP | IMAP over SSL | Безпечне отримання пошти
995 | TCP | POP3 over SSL | Безпечне отримання пошти
3389 | TCP | RDP | Remote Desktop Protocol — віддалений робочий стіл Windows
8080 | TCP | HTTP Alternate | найчастіше використовується для тестування вебпроксі та серверів розробки

## Різниця між портами

**0-1023** (Well-known / System Ports) - це зарезервовані порти  
**1024-49151** (Registered) - це закріплені порти конкретними програмами  
**49152-65535** (Dynamic / Private / Ephemeral Ports) - це тимчасові порти

## Додаткова інформація

HTTP = 80 і HTTPS = 443 тому що так вирішила IANA — вона офіційно закріпила ці номери за цими протоколами. Це просто стандарт і домовленість всього інтернету.

Ephemeral ports — це короткочасні транспортні порти, які операційна система автоматично виділяє для клієнтських програм (наприклад, браузера)

Ні, запустити два сервери на одному порту неможливо, порт може бути зайнятий лише одним процесом.
